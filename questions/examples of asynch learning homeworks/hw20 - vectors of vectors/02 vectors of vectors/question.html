We need to be able to represent data in a data structure that we can treat somewhat like a matrix. Let's see how we can do that with **vectors of vectors**: We can create nested vectors (or **vectors of vectors**). When working with vectors of vectors, it's important to keep track of whether you are working with the *outer vector* or *inner vector*. The vector of vectors will have a **base type**, which is the data type of the individual elements that are stored in the vector. Base types can be `int`, `double`, structs such as the `Rover` struct we've been working with, and any other data type that has been defined.
