There's one more data structure need that we want to address in this course: A lot of engineering data comes in the form of tables or matrices. We need a way to represent that data in C++. Vectors are a good start, but we need to take what we know now and expand it further.
Exercise 1
You have a table of snowfall data (in inches) from Dome #41 on Proxima b in 2023 in the file `snowfall.csv`:
hour 21-Feb-23 22-Feb-23 23-Feb-23 24-Feb-23 25-Feb-23 26-Feb-23 27-Feb-23
1 0.00 0.00 0.00 0.05 0.00 0.00 0.00
2 0.00 1.10 0.00 0.05 0.00 0.00 0.00
3 0.00 0.75 0.00 0.05 0.00 0.00 0.05
... ... ... ... ... ... ... ...
26 0.00 0.00 0.05 0.00 0.00 0.00 0.00

You want to do some data analysis on the snowfall data. What would be the **best** way to represent this data in a C++ program? Hardcode all these values in your program. Read in each value from the file and store it in its own double variable. Read in the values from the file into a series of vectors, one vector for each day. Read in the values from the file into something like a matrix, in whatever way C++ can create "something like a matrix".
