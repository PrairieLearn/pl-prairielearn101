To begin, access the lab's source code by opening the workspace in the "Testing and Implementation" question, which you can find by clicking 'Next Question' on the right.

Overview

In this lab, we are working on a fun typing game "PuppyCaller" that is already in progress. In the game, a farmer is at work with the help of their puppies. Suddenly a loud truck startles the puppies, and they start running away. The farmer must call each puppy by name before they hop the fence! Players play as the farmer, and type puppies' names to call them. Specifically, the game follows this sequence:

  1. The player starts calling a puppy by typing the first letter of its name. This 'selects' the puppy.
  2. The player must type the rest of its name to call it. Any incorrect input (e.g. typing 'a' when the next letter is 'c' is ignored)
  3. Once the player has typed the last character, the puppy is now 'called' and will stop running away. This also unselects the puppy, returning back to step 1.

The game already has a completed console-based user interface and some other functionality, but it is missing a key element: the input handler! In a game, an input handler receives the key(s) the player presses and takes the appropriate action. This is where you come in! You finish the design of the input handler class PuppyCaller based on its specification. Specifically, this includes methods to:

  • Get and set the current selected puppy (useful for tests!)
  • Type the name for the current puppy - corresponding to step 2 and 3 above.
  • Find a puppy whose next character starts with an input character c - helpful for step 1.

These pieces come together in the handleInput method, which receives the input from the player's keyboard, and uses the other methods to implement the logic of the game. Take time to understand the specs of the PuppyCaller and the Puppy class before starting! In the model package, you only need to understand these two classes to complete the lab.
