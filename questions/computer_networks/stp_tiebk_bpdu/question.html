In the spanning tree protocol, when there are two "competing" configurations, four tiebreakers are applied to decide which one to adopt.
Part 1: Tiebreaker on BPDU

Suppose these BPDUs describe the intial configuration of two bridges on a network segment. Which one "wins" the tiebreaker?

STP 802.1d, Config, Flags [none], bridge-id 8000.{{params.id1}}.{{params.port1}}, length 35 message-age 0.00s, max-age 20.00s, hello-time 2.00s, forwarding-delay 15.00s root-id 8000.{{params.root1}}, root-pathcost {{params.rpc1}} STP 802.1d, Config, Flags [none], bridge-id 8000.{{params.id2}}.{{params.port2}}, length 35 message-age 0.00s, max-age 20.00s, hello-time 2.00s, forwarding-delay 15.00s root-id 8000.{{params.root2}}, root-pathcost {{params.rpc2}}

Part 2: Tiebreaker criteria

Which BPDU field was the tiebreaker in the example above?

Lowest root bridge ID Lowest root path cost Lowest sender bridge ID Lowest sender bridge port
